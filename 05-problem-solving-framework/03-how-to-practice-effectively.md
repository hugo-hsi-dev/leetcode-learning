# 5.3 — How to Practice Effectively

## Prerequisites
- All of Sections 1-4

## Section Outline (Detailed)

### 1. The Practice Philosophy
- Quality over quantity
- Understanding > Memorizing
- Spaced repetition
- Active recall

### 2. The 25-Minute Rule
- Try a problem for 25 minutes before looking at hints
- If stuck after 25 minutes:
  - Read the first hint
  - Try for 10 more minutes
  - Read the solution if still stuck
- Time-boxing prevents endless spinning

### 3. Spaced Repetition Schedule
- First solve: Today
- Second solve: Tomorrow
- Third solve: 3 days later
- Fourth solve: 1 week later
- Fifth solve: 2 weeks later
- This builds long-term memory

### 4. The Problem Journal
Keep a record of every problem you solve:

**Entry format:**
```
Date: 2024-01-15
Problem: Two Sum (LeetCode #1)
Pattern: Hash Map
Time: O(n), Space: O(n)
Difficulty: Easy
Time taken: 15 minutes (first time)
Notes: 
- First thought of brute force O(n²)
- Realized hash map gives O(n)
- Remember: check for complement BEFORE adding current to map
```

### 5. How to Use Solutions
- Don't just read — understand
- Re-implement from scratch after reading
- Ask: "Why does this work?"
- Ask: "What's the key insight?"
- Ask: "Could I have thought of this?"

### 6. Active vs. Passive Learning
- **Passive**: Watching videos, reading solutions
- **Active**: Coding from scratch, explaining out loud, teaching

**Active techniques:**
- Explain the solution to yourself (rubber duck debugging)
- Write comments explaining each line
- Re-solve the problem without looking at your previous solution
- Solve variations of the same problem

### 7. Problem Difficulty Progression
1. **Easy problems first**: Build pattern recognition
2. **Medium problems**: Most interview questions are medium
3. **Hard problems**: After comfortable with medium

**Target**: 50-80 easy, 100-150 medium, 20-50 hard

### 8. Curated Problem Lists
Start with curated lists rather than random problems:
- **Blind 75**: Essential problems every candidate should know
- **NeetCode 150**: Expanded list with good coverage
- **Grind 75**: Timed practice sets

### 9. Mock Interviews
- Use Pramp, Interviewing.io, or practice with a friend
- Simulate real conditions: 45 minutes, no IDE, explain out loud
- Focus on communication, not just correctness

### 10. Timed Practice
- Set a timer for 25-30 minutes per problem
- If you can't solve in time, note it and revisit
- Aim to solve most mediums in 20-25 minutes

### 11. What to Do When You Can't Solve
1. Read the solution
2. Understand it completely
3. Close the solution
4. Re-solve from scratch
5. If you get stuck again, re-read and repeat
6. Add to your "revisit" list

### 12. Weekly Schedule Template
**Daily (1-2 hours):**
- 1 new problem
- 2-3 review problems (from previous days)

**Weekly:**
- 5-7 new problems
- 10-15 review problems
- 1 mock interview

**Monthly:**
- Revisit hardest problems from previous months
- Take a practice test

### 13. Red Flags to Watch For
- Solving the same problem multiple times without getting faster
- Memorizing solutions without understanding
- Skipping "easy" problems — they teach patterns
- Not timing yourself
- Only reading solutions, not coding them

### 14. Building Up to Medium
- Start with pattern-based practice
- For each pattern, solve 5-10 problems
- Don't move on until you can solve blind
- Mix patterns after fundamentals

### 15. The "Stuck" Protocol
When you're stuck:
1. Try brute force (always an option)
2. Look at the input — what's the structure?
3. Think: "What if it was sorted?"
4. Think: "Can I use a hash map?"
5. Think: "Is this a DP problem?"
6. If still stuck: read the solution category hint
7. If still stuck: read the full solution

### 16. Common Pitfalls
- Jumping to hard problems too early
- Not reviewing old problems
- Not practicing communication
- Only doing one type of problem
- Giving up too easily / persisting too long

---

## Key Takeaways
1. Use spaced repetition — don't cram
2. Time-box your practice (25-minute rule)
3. Keep a problem journal
4. Re-solve problems from scratch
5. Explain solutions out loud
6. Use curated lists (Blind 75, NeetCode 150)
7. Practice mock interviews before the real thing

## Final Words

Learning to solve LeetCode problems is a skill that improves with deliberate practice. The curriculum you've just read covers the fundamentals — arrays, strings, hash maps, trees, graphs, and the major algorithm patterns.

**Your path forward:**
1. Work through sections 1-5 in order
2. For each topic, read the lesson, then do the practice problems
3. Revisit problems you've solved after a few days
4. Once comfortable with fundamentals, move to Blind 75/NeetCode 150
5. Practice mock interviews 2-3 weeks before your actual interview
